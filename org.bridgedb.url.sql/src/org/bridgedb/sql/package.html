This package is still under development and likely to change without notice.

The reasons this is implemented stand alone rather than in the SimpleGdb are:

  1. This implementation is designed for speed not size
    • All mappings are full mappings not half mappings
    • Columns sizes for exapmple DataSource sysCode may be wider.
  2. This implementation will support different type of mapping (different mapping predicates).
  3. This implementation will support provenance.
  4. There are currently no plans to support AttributeMapper.
  5. Derby will not be hard wired in as a default option.

We do plan on respecting the info table with schemaversion. Older versions will be handled by calling simpleGdbFactory Schemaversion number MUST be corrinated between "SQL" and "SimbpleGdb2 versions.
